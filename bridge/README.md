# MemPalace Bridge for LocalAI Assistant

Connects the Chrome extension to [MemPalace](https://github.com/milla-jovovich/mempalace) for **permanent** memory.

## What it does

The extension **never forgets**:

- **Every action failure** (selector not found, wrong element, timeout) is stored in `wing_action_learning / hall_errors`
- **Every verified success** (correct approach, working selector) is stored in `wing_action_learning / hall_successes`
- **Distilled lessons** ("when X, do Y instead of Z") are generated by the local AI every 10 minutes and stored in `wing_action_learning / hall_lessons`
- **Domain-specific knowledge** (site patterns, UI quirks) goes to `wing_browser_assistant / <domain>`

On every chat message, the extension **recalls** past mistakes, lessons, and domain knowledge from the palace and injects them into the system prompt.

## Setup (already done)

```
# venv + mempalace were installed at:
bridge/.venv/

# Palace initialized at:
~/.mempalace/palace

# Bridge auto-starts via launchd:
~/Library/LaunchAgents/com.localai.mempalace-bridge.plist
```

## Manual control

```bash
# Check status
curl http://127.0.0.1:8765/health

# Start manually (if launchd is not used)
./bridge/start.sh

# Restart via launchd
launchctl kickstart -k gui/$(id -u)/com.localai.mempalace-bridge

# Stop
launchctl unload ~/Library/LaunchAgents/com.localai.mempalace-bridge.plist

# View logs
tail -f /tmp/mempalace-bridge.log
```

## API

| Endpoint | Method | Body | Purpose |
|----------|--------|------|---------|
| `/health` | GET | — | Bridge alive? |
| `/search` | POST | `{query, wing?, room?, limit?}` | Semantic search |
| `/store` | POST | `{wing, room, content, source?, agent?}` | Store one memory |
| `/store-batch` | POST | `{entries: [{wing, room, content, ...}]}` | Store many |

## Extension settings

In **Settings → MemPalace** (already enabled by default):
- **Use MemPalace bridge**: ON
- **Bridge URL**: `http://127.0.0.1:8765`
- **Wing filter**: optional (leave blank for all-wing search)
