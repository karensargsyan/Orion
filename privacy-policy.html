1 Overview

Orion is a Manifest V3 Chrome extension that acts as an AI-powered browser assistant. It is designed with a privacy-first architecture: the extension can operate entirely on your local machine with zero data leaving your device, or optionally connect to cloud AI providers using API keys that you supply and control.

Core principle: Orion has no telemetry, no analytics, no tracking, and no data collection infrastructure of any kind. The developer does not operate servers that receive your data. The extension runs entirely within your browser.

2 Data Collection & Types

Orion does not collect, transmit, or store any user data on external servers. All data is stored locally in your browser using Chrome's Storage API and IndexedDB. The following table discloses every category of data that Orion processes, how it is stored, and whether it is ever transmitted externally.

No hidden collection: Orion does not collect browsing history, cookies, bookmarks, or any data from tabs outside the Orion tab group. There is no background data harvesting, no usage analytics, no telemetry, and no advertising integrations.

3 How Data Is Used

Each type of data Orion processes is used for a specific, user-facing purpose:

  • Page content (URLs, text, forms): Sent to your chosen AI provider (local or cloud) to generate contextual assistance, answer questions, fill forms, or automate browser actions that you request.
  • Conversation history: Stored locally to maintain chat context across sessions so you can continue previous conversations.
  • Vault entries (credentials, addresses, cards): Stored locally with AES-256 encryption. Used solely to auto-fill forms when you request it.
  • API keys: Stored locally. Used solely to authenticate requests to AI provider APIs (Google Gemini, OpenAI, Anthropic) that you configure.
  • User settings: Stored locally. Used to configure the extension's behavior according to your preferences.
  • Learning data (optional): When you enable AI learning, DOM snapshots and action outcomes are recorded locally to improve the AI's accuracy on sites you frequently use. This data never leaves your device.
  • Screenshots (optional): When vision mode is enabled, viewport screenshots are captured and sent to your AI provider as part of the prompt. Screenshots are not stored persistently.

Data is never used for advertising, profiling, marketing, or any purpose other than providing the features described above directly to you.

4 Local vs Cloud Mode

Orion supports two operating modes. You choose which one to use.

Local Mode (LM Studio / Ollama)

  • AI runs entirely on your machine
  • Zero data leaves your device
  • Connects only to localhost / 127.0.0.1
  • No internet connection required for AI
  • Maximum privacy guarantee

Cloud Mode (Optional)

  • Uses your own API keys
  • Supported: Google Gemini, OpenAI, Anthropic
  • Only prompt content is transmitted
  • Keys stored locally, never sent to the developer
  • Subject to each provider's privacy policy
Important: When using cloud mode, the content of your prompts and the page context you provide are sent to the selected AI provider's API. This is a direct connection between your browser and that provider -- Orion does not proxy, log, or intercept this traffic. Review the privacy policy of your chosen provider (Google, OpenAI, or Anthropic) to understand how they handle API requests.

5 Permissions Explained

Chrome extensions must declare the permissions they require. Below is every permission Orion requests, along with why it is needed and what data it accesses. No permission is used beyond its stated purpose.

Permission Why Orion Needs It
storage Save your settings, preferences, and conversation history locally in Chrome's storage. No data is transmitted externally.
unlimitedStorage Allow vault data, conversation history, and learning data to exceed Chrome's default 10 MB storage quota. All data remains local.
tabs Read tab titles and URLs to provide context-aware assistance. Only accesses tabs within the Orion tab group -- your other tabs are never read.
tabGroups Create and manage a dedicated tab group so Orion only interacts with tabs you explicitly assign to it.
activeTab Access the content of the currently active tab when you invoke Orion, scoped to only the tab you are viewing.
sidePanel Display Orion's chat interface in Chrome's built-in side panel.
scripting Inject content scripts to read page content or perform actions (e.g., form filling, element clicking) on pages within the Orion tab group. Scripts are bundled with the extension -- no remote code is loaded.
webNavigation Detect when pages finish loading within the tab group so Orion can update page context after navigation events.
alarms Schedule periodic background tasks such as monitoring checks at intervals you configure in Settings.
offscreen Create offscreen documents for operations that require a DOM context (e.g., processing screenshots, clipboard operations) without visible windows.
debugger Use the Chrome DevTools Protocol (CDP) to perform reliable, trusted browser interactions (clicks, typing, scrolling) on pages within the Orion tab group. This permission is used exclusively for browser automation that you initiate -- never for monitoring, data collection, or surveillance.
contextMenus Add right-click menu options so you can send selected text or page content to Orion for processing.
notifications Show browser notifications when automated tasks complete or when the extension needs your attention (e.g., action confirmation). Notifications are generated locally and do not transmit data.

6 Storage & Encryption

All persistent data is stored locally within your browser using Chrome's Storage API and IndexedDB. Nothing is stored on remote servers.

  • Settings and preferences: Saved in Chrome's local storage on your device.
  • API keys: Stored in Chrome's local storage on your device. They are sent only to the respective AI provider's API endpoint when you make a request -- never to the developer or any other third party.
  • Vault data (credentials, addresses, payment cards, identities): Encrypted using AES-GCM 256-bit authenticated encryption via the Web Crypto API. The encryption key is derived from your user-defined PIN using PBKDF2 key derivation with a unique salt. Your PIN and encryption key are never stored in plaintext.
  • Conversation history: Stored locally in IndexedDB. Not encrypted at rest, but never transmitted externally.
  • Learning data: Stored locally in IndexedDB. Never transmitted externally.
Encryption details: The vault uses AES-GCM authenticated encryption, which provides both confidentiality and integrity. Each entry is encrypted individually. Without your PIN, the encrypted data is computationally infeasible to recover. The vault auto-locks after a configurable idle timeout.

7 Third-Party Services

Orion can optionally connect to the following services. All connections are initiated by you and configured with your own credentials. The developer has no access to your accounts or API keys.

Local Services (no external network traffic)

  • LM Studio / Ollama -- Local AI language model server. Runs entirely on your machine at localhost. Zero data leaves your device.
  • MemPalace -- Local vector embedding service for semantic search. Runs on your machine.
  • Whisper -- Local speech-to-text transcription server. Runs on your machine.

Cloud AI Providers (optional, user-configured)

When you configure a cloud provider, Orion sends your AI prompts (which may include page content from the active Orion tab) directly to that provider's API. The developer does not proxy, log, or have access to this traffic.

  • Google Gemini API -- Prompt content sent to Google's API servers. Subject to Google's Gemini API Terms of Service and Google Privacy Policy.
  • OpenAI API -- Prompt content sent to OpenAI's API servers. Subject to OpenAI's Terms of Use and OpenAI Privacy Policy.
  • Anthropic API -- Prompt content sent to Anthropic's API servers. Subject to Anthropic's Terms of Service and Anthropic Privacy Policy.

Optional: Telegram Bot

If you configure a Telegram bot token in Settings, Orion communicates with the Telegram Bot API to receive your messages and send responses. Only messages you send to your own bot are processed. The developer has no access to your bot token or messages. Subject to Telegram's Privacy Policy.

8 Data Sharing

Orion does not sell, rent, trade, or disclose your data to any third party. Specifically:

  • No data is sold to data brokers or any other parties.
  • No data is shared with advertising networks or marketing services.
  • No data is used for targeted advertising or user profiling.
  • No data is shared with the developer's own servers (the developer operates no data-receiving servers).
  • No data is transferred to third parties except the AI provider APIs you explicitly configure and invoke.
  • No data is used for creditworthiness or lending purposes.

The only external data transfer occurs when you use a cloud AI provider: the prompt content (which may include page context from your active Orion tab) is sent directly from your browser to that provider's API. This transfer is initiated by you, uses HTTPS encryption in transit, and is governed by the provider's own privacy policy.

9 Tab Isolation

Despite holding the <all_urls> host permission, Orion enforces strict self-imposed boundaries on which tabs it accesses:

  • Orion creates a dedicated tab group with a distinct color in your browser.
  • It only reads content from and interacts with tabs that are inside this tab group.
  • Tabs are only added to the group when you explicitly open them through Orion or manually add them.
  • Your other browser tabs, bookmarks, history, and browsing activity outside the tab group are never accessed.
  • The side panel only activates on tabs within the Orion tab group.
In practice: If you have 50 tabs open and only 3 are in the Orion tab group, Orion can only see those 3 tabs. The other 47 are invisible to it.

10 Data Retention & Deletion

All data stored by Orion is local to your browser and remains under your direct control.

  • Retention period: Data persists in your browser storage until you explicitly delete it or uninstall the extension. There are no automatic expiration periods.
  • Manual deletion: You can clear conversation history, vault entries, learning data, and settings at any time through the extension's interface.
  • Uninstall: Removing the extension from Chrome completely and permanently deletes all locally stored data associated with Orion.
  • No remote backup: The developer does not maintain copies or backups of your data. Once deleted locally, it is permanently gone.
  • Cloud provider data: If you use cloud AI providers, the data sent in API requests may be retained by those providers according to their own retention policies. Consult each provider's privacy policy for details.

11 Your Rights (GDPR, CCPA)

Since Orion stores all data locally on your device and the developer collects no data on external servers, you have complete, direct control over your information at all times.

General Rights

  • Access: All your data is stored in Chrome's local storage and IndexedDB. You can view it through the extension interface, or inspect it using browser developer tools (F12 > Application > Storage).
  • Deletion: Delete individual items (conversations, vault entries) through the extension's UI, or uninstall the extension to remove all data at once.
  • Portability: Export vault data or conversation history through the extension's export features.
  • Control: Choose between local-only mode (zero external connections) or cloud mode with providers of your choice. Every feature that sends data externally requires your explicit configuration.
  • Transparency: The full source code is available on GitHub for independent audit and verification of all privacy claims.

For Users in the European Economic Area (GDPR)

Because Orion does not collect or process personal data on external servers, most GDPR obligations related to data controller responsibilities do not apply to the developer. However, you retain all rights under the GDPR, including the right to access, rectification, erasure, data portability, and the right to object to processing. Since all data is stored locally on your device, you can exercise these rights directly without needing to contact the developer. If you use cloud AI providers, those providers act as independent data processors and their GDPR compliance statements apply.

For Users in California (CCPA/CPRA)

Orion does not sell personal information. The developer does not collect, use, or share personal information for cross-context behavioral advertising. Since no personal data is collected on external servers, there is no data to disclose, delete, or opt out of on the developer's side. If you use cloud AI providers, consult their respective CCPA/CPRA disclosures.

12 Security Measures

Orion implements multiple security layers to protect your data:

  • Manifest V3: Built on Chrome's latest extension security architecture with restricted APIs and enhanced permission controls.
  • AES-GCM encryption at rest: Vault data (credentials, payment cards, personal information) is encrypted with 256-bit keys derived from your PIN using PBKDF2.
  • HTTPS in transit: All connections to cloud AI provider APIs use encrypted HTTPS/TLS. No user data is ever sent over unencrypted HTTP.
  • Content Security Policy: Strict CSP prevents injection attacks and unauthorized script execution within the extension.
  • Tab group isolation: Extension only accesses tabs you explicitly assign to the Orion group, not your entire browser.
  • No remote code execution: All JavaScript is bundled with the extension package. No external scripts are loaded or executed at runtime.
  • Local mode option: Use local AI models for zero-trust, air-gapped operation with no internet requirement.
  • Auto-lock: The vault automatically locks after a configurable idle timeout, requiring your PIN to unlock.

13 Children's Privacy

Orion is not directed at children under the age of 13. The developer does not knowingly collect personal information from children under 13. Since Orion stores all data locally on the user's device and the developer has no access to any user data, there is no mechanism through which children's data could be collected by the developer.

If you believe a child under 13 has used the extension with a cloud AI provider, please contact the respective AI provider regarding their handling of that data. You may also contact the developer at the email address below with any concerns.

14 Google API Limited Use Disclosure

Limited Use Compliance: Orion's use and transfer to any other app of information received from Google APIs will adhere to the Chrome Web Store User Data Policy, including the Limited Use requirements.

Specifically, Orion:

  • Only uses data obtained through Chrome APIs for the single purpose of providing the user-facing features described in this policy and in the Chrome Web Store listing.
  • Does not transfer user data to third parties except to provide or improve user-facing features (AI provider API calls you configure), for security purposes, or to comply with applicable laws.
  • Does not use or transfer user data to serve advertising.
  • Does not allow humans to read user data, except with the user's affirmative consent, for security investigations, or to comply with applicable laws.
  • Does not sell user data.

15 Compliance

  • Chrome Web Store Developer Program Policies: Orion complies with all applicable Chrome Web Store Developer Program Policies, including User Data, Limited Use, and permissions justification requirements.
  • Manifest V3 requirements: The extension is built on Manifest V3 with no use of remotely hosted code.
  • Data minimization: Only essential data (page context for AI prompts) is processed, and only when you invoke the assistant or enable specific features.
  • Prominent disclosure: The extension's side panel UI clearly indicates when data is being sent to a cloud AI provider. Local mode connections are visually distinguished from cloud connections.
  • User consent: Cloud AI features require explicit configuration with your own API keys. No data is sent to external services without your deliberate action.
  • Sensitive permissions: The debugger permission is used solely for reliable browser automation within your explicitly assigned tab group. The <all_urls> host permission is constrained at runtime to only tabs in the Orion tab group.
Cloud provider compliance: If you use cloud AI providers (Gemini, OpenAI, Anthropic), their respective privacy policies, GDPR compliance, and data handling obligations apply to the data you send through their APIs. Review each provider's documentation if you operate under specific regulatory requirements.
