📎 File Upload Test Page

Use this page to test Orion's file upload feature with comprehensive debug output.

📋 Test Instructions

  1. Click the Orion extension icon in your browser toolbar
  2. Open the sidepanel
  3. Click the 📎 paperclip icon at the bottom of the chat
  4. Select the test.txt file you created (see below)
  5. Type a message like: "analyze this file" or "what's in the file?"
  6. Press Send
  7. Check for [DEBUG] messages in the chat
ℹ️ Note: The debug messages will appear as colored banners in the chat area showing exactly what's happening at each step.

✅ Expected Debug Messages

If file upload works correctly, you should see these debug messages in order:

[DEBUG] File selected: test.txt (XXX bytes) [DEBUG] File loaded: test.txt (XXX chars) [DEBUG] Preview bar shown [DEBUG] Preparing to send file: test.txt (XXX chars) [DEBUG] File content prepended. Total text: XXX chars [DEBUG] File card added to chat bubble [DEBUG] Message sent to AI (XXX chars) [DEBUG] Service worker received file: test.txt (XXX chars)
⚠️ If any message is missing: That's where the bug is! Check the browser console (F12) for detailed logs starting with [FILE UPLOAD], [SEND MESSAGE], or [AI CHAT].

🐛 Common Issues to Check

  • No "File selected" message: File picker not working
  • No "File loaded" message: FileReader failed
  • No "Preview bar shown": Preview UI elements missing/broken
  • "WARNING: fileName exists but no fileContext": State cleared too early
  • No "File card added": Card creation or append failed
  • No "Service worker received": Message format wrong or not reaching backend

🔍 Additional Debugging

Open browser console (F12) and filter by:

  • [FILE UPLOAD] - File loading process
  • [SEND MESSAGE] - Message preparation and sending
  • [AI CHAT] - Service worker reception
  • [FILE CARD] - File card UI creation
💡 Tip: Open console BEFORE uploading the file to capture all log messages from the start.
